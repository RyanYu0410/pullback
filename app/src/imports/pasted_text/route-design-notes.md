1. Pace Selection — “How Long Can You Hold It?”
Screen feeling

Instead of asking for age first, ask for a felt capacity.
Age can be optional later, but the emotional entry should be about the user’s own rhythm.

On-screen copy

How long can you usually hold a line?

Options:

5 min
10 min
15 min
20 min
I choose my own

Small text:
No perfect number. Just your pace tonight.

Why this is stronger

It avoids sounding like a test.
It gives the user ownership from the beginning.

2. Intention Note — “Write What You Don’t Want to Lose”
Screen feeling

A floating paper note appears.
It should feel slightly vulnerable, like a small object placed into the interface.

On-screen copy

What do you not want to lose?

Placeholder examples:

“the first 20 minutes after dinner”
“reply and leave”
“start homework before I disappear”
“just check one thing”
“don’t lose tonight”

CTA:
Keep this

Interaction

The user writes one short note.
After submitting, the note folds slightly or pins itself to the top of the line.

Emotional purpose

This is where ownership becomes real.
The app is not creating a task.
The user is protecting a small personal intention.

3. Tether Ritual — “Pull the Line Tight”
Screen feeling

The loose line now drops down the screen.
It curves, bends, and floats away from the note.

The user must drag the bottom of the line downward to tighten it.

On-screen copy

Pull the line tight.

Small text:
This is your route.

Interaction

As the user drags:

the line straightens slightly
the note becomes more stable
the silhouette becomes connected to the path
the line gives a soft elastic response
a quiet pulse appears when the line is secured
Success copy

Still yours.

Why this is more intense

The user physically performs the core metaphor.
They are not tapping “next.”
They are reclaiming control with a gesture.

4. Path Builder — “Place What Belongs”
Screen feeling

The user now sees a winding path.
The note is pinned at the top.
The path curves downward, with empty points along the way.

Instead of feeling like a checklist, it should feel like drawing a route through a fragile space.

On-screen copy

Place what belongs on this line.

Components

The user can drag two types of objects onto the path:

A. App blocks

These represent tools or stops.

Examples:

Messages
Notes
Calendar
Music
Safari
School app
B. Emotional route cards

These are more important than the apps.

Card meanings

White — Note
The intention anchor.
Meaning: This is what I am protecting.

Green — Steady
A safe or useful stop.
Meaning: This helps me stay on route.

Yellow — Pause
A controlled break.
Meaning: This is a pause, not a disappearance.

Red — Pull
A risky point.
Meaning: This may stretch the line.

Blue — Return
A recovery point.
Meaning: This is where I can come back.

Why this is stronger

The user is not simply organizing apps.
They are designing the emotional behavior of the route.

They are saying:
“This part may pull me.”
“This part helps me.”
“This is where I return.”

That creates much more ownership.

5. Route Ready — “The Note Is Attached”
Screen feeling

The completed path is shown as a full object.
The note is attached to the line.
The app blocks and cards sit along the route.

On-screen copy

The note is attached.
Start before it loosens.

CTA options:

Start now
Save as widget
Emotional purpose

This screen should feel like the user has prepared something fragile and important.
The route is not just saved.
It is held.

6. Widget Creation — “Keep It Near the Door”
Screen feeling

The widget page should feel like placing the note near the entrance of the phone.

The widget is not a dashboard.
It is a small visible tether on the home screen.

On-screen copy

Keep this line near the door.

Widget options:

Small: note + line icon + Start
Medium: note + route preview + Start
Minimal: only line state + Start
Best widget copy

Don’t lose this.
or
Start with the note.

Why this matters

The widget becomes the threshold between unconscious phone entry and intentional entry.

The home screen is no longer just a field of apps.
It now contains a visible reminder of what the user meant to keep.

Main Daily Flow
7. Start from Widget — “Crossing the Threshold”
User action

The user taps the widget.

System response

The app does not open into a heavy dashboard.
Instead:

the route starts
the timer begins
the Live Activity appears
the note becomes active
the line state becomes visible
Microcopy

Line active.
Note in hand.

Emotional purpose

Starting should feel immediate and physical.
The user has crossed into the phone while holding something.

8. Live Activity — “The Line Lives Outside the App”
Screen feeling

The Live Activity is the emotional center of the product.

It should show:

the note preview
the line state
time remaining
one action: Return
Normal state copy

Line steady.
Still yours.

Visual state

The line is thin, calm, and connected.
The note is stable.

Why this is stronger

The product does not live inside the app.
It lives at the system layer, where the user can still feel the route while using the phone.

9. Drift State — “The Line Begins to Loosen”
Trigger

This can happen through elapsed time, route stage, user self-trigger, or prototype scenario.

Screen feeling

The Live Activity changes subtly but emotionally:

the line becomes wavier
the note lifts slightly away from the line
the line becomes thinner
a small gap appears near one connection point
the background becomes slightly colder
the red Pull card may glow faintly
On-screen copy

The line is loosening.

Secondary options:

You are being pulled.
The note is slipping.
You can still return.
Actions
Return
Pause
Let it go
Stay 3 min
Why this is intense

The app does not say “You failed.”
It makes the user feel that something precious is becoming unstable.

That creates urgency without shame.

10. Return Moment — “Pull It Back”
User action

The user taps Return.

Interaction

The user may also drag the loose line back into place.

The return action should echo the onboarding gesture.

Visual response
the line tightens
the note reconnects
the red glow fades
the blue Return card activates
the path recenters
a quiet pulse moves through the line
Copy

You pulled it back.

Other options:

Note kept.
Line restored.
Still yours.
Not lost.
You came back in time.
Emotional purpose

This is the victory.

Not a badge.
Not confetti.
Not a productivity score.

The victory is the relief of recovery.

End Flow
11. Route Ending — “What Happened to the Line?”
Screen feeling

The session ends with the line shown as a record.

The path may include:

stable segments
loosened segments
return points
note still attached or partially recovered
On-screen copy

What happened to the line?

Reflection options:

Held steady
Loosened once
Came back in time
Got carried away

CTA:
Save

Why this is stronger

The question is not “Did you succeed?”
It is “What happened?”

That keeps the product reflective instead of judgmental.

12. Victory Record — “Not Perfect, Not Lost”
Screen feeling

A saved record appears as a quiet card.

It should feel like a preserved trace, not a performance report.

Content
original note
route name
duration
return moment
reflection tag
line state
Strong victory copy

Not perfect. Not lost.

Other options:

One return saved the route.
The note stayed with you.
You came back before it broke.
Still yours.
Emotional purpose

This is where the product’s philosophy becomes clear:

Success is not perfect restraint.
Success is returning before the intention is gone.
